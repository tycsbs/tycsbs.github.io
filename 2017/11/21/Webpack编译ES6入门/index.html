什么是webpack

webpack是一个用以对现代Javascript应用程序模块打包工具，它和gulp、grunt的不同之处在于webpack提供了一套前端模块化开发的解决法案，把各种资源作为模块来处理，包括模块的加载、预处理、压缩、打包等。

webpack的优点

  1. 模块化，在开发过程中，我们可以把程序代码按照不同职能细化成单个模块进行开发，又可以在运行时通过webpack将各个模块进行合并压缩，提高开发效率和方便代码维护。
  2. 代码统一性，前端技术发展日新月异，浏览器对代码的解析能力也千差万别，而webpack可以通过配置指定的模块和加载器，将代码解析，最终打包成一套兼容各现代浏览器的程序。
  3. 资源处理，webpack能够自动对代码进行管理和检测，能够对资源进行压缩、打包、样式预处理、图片资源转base64等。

webpack 使用 （基于webpack2.0）

光说不练假把式，webpack我们提供了一套模块开发的解决方案，如果想一下子全部弄明白肯定是有些困难的，因此，路要一步一步走，我们先从零搭建一个webpack的运行开发环境，一步一步开始webpack的学习

初体验

  1. 初始化 新建一个文件夹，取名叫webpackDemo,通过控制台或者git bash定位到当前文件下面，执行命令：
npm init //淘宝镜像用 cnpm init

然后一路enter,初始化后，会在该文件夹下面生成一个package.json文件，然后，再次在命令行内执行：

npm install //淘宝镜像用 cnpm install
  1. 项目结构搭建 在webpackDemo文件夹下，新建两个文件夹和webpack.config.js，分别叫做src和dist，src文件夹下面用来放我们开发的文件，dist文件夹用来放webpack打包后的文件。webpack.config.js是webpack打包配置的文件，需要注意的是，它的名字是固定的，不能叫做别的名字。然后src文件夹下面我们再根据项目结构，分别新建js、css等资源文件夹,文件最终结构为：
